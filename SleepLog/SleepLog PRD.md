# SleepLog PRD (Product Requirements Document) 

## 1. 개요 

**SleepLog**는 Gemini AI를 사용자가 복잡한 설정 없이 매일의 수면 시간을 간단히 기록하고, 자신의 수면 패턴을 직관적으로 파악할 수 있도록 돕는 ios 앱이다. 

## 2. 타겟 사용자 
### 주 사용자 
-불규칙한 수면 패턴으로 생활 리듬에 영향을 받는 것을 개선하고자 하는 모든 스마트폰 사용자.

##3. 핵심 기능 (MVP) 
### P1 - 필수 기능 
|  기능  |  설명  |  Gemini AI 활용  |   참고 사항/레퍼런스 |
|———————————————————-|
| 수면 기록 입력 | 사용자가 ‘취침 시간’과 ‘기상 시간’을 하나의 화면에서 직관적으로 입력할 수 있어야 한다.  |  Sleep Cycle의 시간 설정 UI 참고 
| 총 수면 시간 자동 계산 | 입력된 취침 시간과 기상 시간을 바탕으로 총 수면 시간을 자동으로 계산하여 사용자에게 표시되어야 한다. (예: 7시간 30분) |    | 
|수면 기록 저장/수정/삭제 | 입력된 수면 기록은 날짜별로 저장되어야 하며, 사용자는 언제든지 기록을 수정하거나 삭제할 수 있어야 한다. | 
|수면 기록 목록 뷰 | 저장된 모든 수면 기록을 최신순으로 List 형태로 표시해야 한다. (날짜, 취침 시간, 기상 시간, 총 수면 시간 포함) | Samsung Health의 기록 목록 참고.
|수면 메인 화면| 오늘의 수면 기록 요약 (혹은 수면 점수), 지난 7일간의 평균 수면 시간, 최근 기록 등 핵심 정보를 요약하여 표시해야 한다.|' 수면 점수' 제공 구조 참고.

##4. 비기능 요구 사항 
|항목|내용| 
|성능|수면 기록 입력 및 계산은 1초 이내에 처리되어야 한다. |
|사용성(UX/UI) | UI는 간결하고 직관적이어야 하며, 불필요한 단계 없이 빠르게 기록할 수 있어야 한다. | 
|아키텍처| MVVM (Model-View-ViewModel) 디자인 패턴을 적용하여 개발해야 한다. | 
|호환성| iOS/Android 모바일 환경에서 안정적으로 동작해야 한다.| 


##5. Gemin API 통합 전략 
### 프롬프트 전략 
- **시스템 프롬프트**: 긍정적, 격려하는 톤 

## 6. UI/UX 원칙 

###  디자인 시스템 
- **색상**: 다크 기반 (눈이 편하게) 
- “”타이포””: SF Pro Rounded 

### 핵심 화면 
1. **홈**: 수면 대시보드(수면 시작 버튼 -> 수면 끄기 버튼, 수면 통계표) 
2. **수면 시작 화면**: 버튼 클릭시에 수면 모드 실시
3. **수면 기록 화면**: 차트 형태로 잠든 시간, 일어난 시간 기록 
4. **통계 화면**: 일/주/월 그래프
